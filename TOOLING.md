# Tooling Context

This proof ledger is informed by hands-on use of multiple AI coding and agentic development tools.

## Tools used

- Claude Code
- OpenAI Codex CLI
- Devin
- VS Code with GitHub Copilot
- Cursor

## Why this matters

The private workflow is not designed around a single vendor interface.

It is informed by practical use of multiple coding-agent surfaces and focuses on portable workflow concepts:

- structured backlog issues
- compact handoff artifacts
- role separation
- validation before publication
- reviewer/judge separation
- token- and context-efficiency
- private/public proof boundaries

## Claim boundary

This file does not claim formal benchmarking across these tools.

It records hands-on exposure that shaped the workflow design.

The claims in this repository should be evaluated through `CLAIMS.md`, `EVIDENCE.md`, the sanitized case studies, and any future run-proof artifacts.
